Réseaux

  • ×

    SSH (Secure Shell) :

    Protocole sécurisé pour l’administration à distance des équipements réseau.

  • ×

    Telnet :

    Protocole permettant un accès distant aux équipements réseau, mais sans chiffrement (peu sécurisé).

  • ×

    SNMP (Simple Network Management Protocol) :

    Permet de surveiller et gérer les équipements réseau.

  • ×

    (Dynamic Host Configuration Protocol) :

    Attribue automatiquement des adresses IP aux appareils sur un réseau.

  • ×

    DNS (Domain Name System) :

    Convertit les noms de domaine (ex : google.com) en adresses IP.

  • ×

    IMAP (Internet Message Access Protocol) :

    Protocole plus avancé que POP3, permettant de gérer ses mails directement sur le serveur.

  • ×

    POP3 (Post Office Protocol v3) :

    Permet de récupérer des courriels depuis un serveur vers un client de messagerie.

  • ×

    SMTP (Simple Mail Transfer Protocol) :

    Protocole permettant l’envoi de courriels entre serveurs.

  • ×

    SFTP (Secure File Transfer Protocol) :

    Variante sécurisée de FTP, utilisant SSH pour sécuriser les transferts de fichiers.

  • ×

    FTP (File Transfer Protocol) :

    Protocole permettant le transfert de fichiers entre un client et un serveur.

  • ×

    HTTPS (HTTP Secure) :

    Version sécurisée de HTTP, utilisant SSL/TLS pour le chiffrement des communications.

  • ×

    HTTP (HyperText Transfer Protocol) :

    Utilisé pour la communication entre les navigateurs web et les serveurs.

  • ×

    UDP (User Datagram Protocol) :

    Protocole rapide mais non fiable, utilisé pour les applications où la rapidité est plus importante que la fiabilité (ex : streaming, jeux en ligne).

  • ×

    TCP (Transmission Control Protocol) :

    Protocole fiable qui assure l’envoi correct des données avec contrôle d’erreur et accusés de réception.

  • ×

    ICMP (Internet Control Message Protocol) :

    Utilisé pour diagnostiquer les problèmes de réseau, notamment avec des outils comme ping et traceroute.

  • ×

    IPv6 (Internet Protocol version 6) :

    Nouvelle version d'IPv4, utilisant des adresses 128 bits pour résoudre le problème d’épuisement des adresses.

  • ×

    IPv4 (Internet Protocol version 4) :

    Protocole principal d’adressage et de routage des paquets sur Internet, utilisant des adresses 32 bits.

  • ×

    ARP (Address Resolution Protocol) :

    Permet de retrouver l'adresse MAC correspondant à une adresse IP sur un réseau local.

  • ×

    Ethernet (CSMA/CD) :

    Règle qui gère l'accès aux réseaux filaires, en évitant les collisions de données.

  • ×

    Wi-Fi (IEEE 802.11) :

    Ensemble de protocoles pour les réseaux sans fil, permettant la connexion entre des appareils via des ondes radio.

  • ×

    Ethernet (IEEE 802.3) :

    Norme qui définit le câblage, les vitesses et la manière dont les données sont transmises sur un réseau filaire.

  • ×

    CAPWAP (Control and Provisioning of Wireless Access Points) :

    Permet aux contrôleurs Wi-Fi de gérer à distance plusieurs points d’accès sans fil.

  • ×

    WPA / WPA2 / WPA3 :

    Protocoles de sécurité pour les réseaux Wi-Fi, protégeant les connexions contre les attaques.

  • ×

    ACL (Access Control List) :

    Définit des règles de filtrage du trafic sur un routeur ou un pare-feu.

  • ×

    NAT (Network Address Translation) :

    Permet de traduire des adresses IP privées en IP publiques pour accéder à Internet.

  • ×

    Syslog :

    Collecte et stocke les logs d’événements des équipements réseau.

  • ×

    NTP (Network Time Protocol) :

    Synchronise l’heure des équipements réseau avec une horloge de référence.

  • ×

    LLDP (Link Layer Discovery Protocol) :

    Alternative à CDP, fonctionne avec des équipements non Cisco.

  • ×

    CDP (Cisco Discovery Protocol) :

    Permet aux équipements Cisco de s’identifier entre eux.

  • ×

    HSRP (Hot Standby Router Protocol) :

    Protocole de redondance permettant d’assurer la haute disponibilité des routeurs.

  • ×

    EIGRP (Enhanced Interior Gateway Routing Protocol) :

    Protocole Cisco optimisé pour un meilleur équilibre entre rapidité et consommation de ressources.

  • ×

    OSPF (Open Shortest Path First) :

    Protocole de routage avancé basé sur l’état des liens, plus rapide et plus efficace que RIP.

  • ×

    RIP (Routing Information Protocol) :

    Protocole de routage basé sur la distance, simple mais limité (utilise le "nombre de sauts" pour choisir la meilleure route).

  • ×

    EtherChannel (LACP / PAgP) :

    Regroupe plusieurs liaisons physiques entre commutateurs pour augmenter la bande passante et la redondance.

  • ×

    MSTP (Multiple Spanning Tree Protocol) :

    Permet d’optimiser la gestion de plusieurs VLANs en associant plusieurs instances STP.

  • ×

    RSTP (Rapid Spanning Tree Protocol) :

    Version améliorée de STP, qui réduit le temps de convergence en cas de changement de topologie.

  • ×

    STP (Spanning Tree Protocol) :

    Empêche les boucles dans un réseau commuté en désactivant certains liens redondants.

  • ×

    Trunking (802.1Q) :

    Permet de transporter plusieurs VLANs sur un même lien physique.

  • ×

    VLAN (Virtual LAN) :

    Technologie permettant de segmenter un réseau physique en plusieurs réseaux logiques.

  • ×

    BPDU (Bridge Protocol Data Unit) :

    Messages échangés entre les commutateurs dans le cadre du Spanning Tree Protocol (STP). Les BPDUs permettent aux commutateurs de détecter les boucles dans le réseau et d’élire un root bridge (pont racine). Ils contiennent des informations comme les priorités des commutateurs et les coûts des liens, afin de déterminer la meilleure topologie sans boucle.

Informatique

  • ×

    JavaScript :

    JavaScript est un langage de programmation interprété principalement utilisé pour le développement web. Il permet d'ajouter de l'interactivité aux pages web, de manipuler le DOM (Document Object Model) et de gérer des requêtes asynchrones via AJAX. C'est un langage orienté objet basé sur les prototypes, largement utilisé aussi bien côté client que côté serveur grâce à des environnements comme Node.js.

  • ×

    PHP :

    PHP (Hypertext Preprocessor) est un langage de programmation côté serveur principalement utilisé pour le développement web. Il permet de générer des pages dynamiques en interagissant avec des bases de données et des formulaires. Son intégration facile avec HTML et sa compatibilité avec de nombreux systèmes de gestion de bases de données en font un choix populaire pour les sites web et les applications web.

  • ×

    Python :

    Python est un langage de programmation interprété, polyvalent et facile à apprendre, largement utilisé pour le développement web, l'automatisation, la science des données, l'intelligence artificielle et la cybersécurité. Il est apprécié pour sa syntaxe simple et lisible, ainsi que pour son vaste écosystème de bibliothèques et frameworks facilitant le développement rapide d'applications.

  • ×

    HTML :

    HTML (HyperText Markup Language) est un langage de balisage utilisé pour structurer le contenu des pages web. Il permet de définir les éléments d'une page tels que les titres, les paragraphes, les images, les liens et les formulaires. HTML fonctionne en combinaison avec CSS et JavaScript pour créer des interfaces utilisateur interactives et est la base de tout document web.

  • ×

    CSS :

    CSS (Cascading Style Sheets) est un langage de feuille de style utilisé pour contrôler l’apparence et la mise en page des éléments HTML. Il permet de définir des styles tels que les couleurs, les polices, les marges et les animations. CSS est essentiel pour créer des interfaces attrayantes et responsives, et il peut être complété par des préprocesseurs comme SASS ou LESS pour une gestion avancée du design.

  • ×

    C++ :

    C++ est un langage de programmation compilé et orienté objet, basé sur C, utilisé pour le développement d'applications à haute performance, de jeux vidéo, de logiciels système et d’applications embarquées. Il offre un contrôle précis de la mémoire et des ressources, ainsi qu’une grande flexibilité grâce à ses paradigmes de programmation procédurale, orientée objet et générique.

  • ×

    SQLite :

    SQLite est un moteur de base de données relationnelle léger, intégré et sans serveur. Il est utilisé dans des applications mobiles, embarquées et des logiciels nécessitant une gestion efficace des données sans nécessiter une infrastructure complexe. Fonctionnant avec le langage SQL, il permet de stocker et manipuler des bases de données en local avec un faible encombrement mémoire.

  • ×

    Linux :

    Linux est un système d’exploitation open source basé sur le noyau Linux, utilisé pour les serveurs, les systèmes embarqués et les ordinateurs personnels. Il est apprécié pour sa stabilité, sa sécurité et sa flexibilité, permettant aux utilisateurs d’exécuter des scripts shell, de gérer des processus et d’administrer des réseaux. Il est disponible sous différentes distributions adaptées à divers besoins, allant des environnements grand public aux infrastructures critiques.
